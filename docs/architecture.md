# RAG知识库问答系统 - 架构设计文档

## 1. 系统概述

本系统基于检索增强生成（Retrieval-Augmented Generation, RAG）技术，构建企业级知识库问答系统。系统支持多格式文档的智能检索与问答，具备高准确率、低延迟、可扩展等特点。

## 2. 核心架构

### 2.1 整体架构

系统采用分层架构设计，从上到下依次为：

1. **用户界面层**：提供Web交互界面（Streamlit）和RESTful API接口（FastAPI）
2. **应用服务层**：处理用户请求，管理会话状态
3. **RAG核心层**：实现检索增强生成的核心逻辑
4. **数据存储层**：向量数据库、文档存储、会话存储
5. **文档处理层**：文档解析、切分、向量化

### 2.2 核心流程

```
用户问题 → 查询改写 → 检索 → 重排序 → 上下文构建 → 答案生成 → 返回结果
```

## 3. 核心模块设计

### 3.1 文档处理模块

**功能**：
- 支持多格式文档解析（PDF、Word、Markdown、TXT）
- 文档切分（语义切分 + 滑动窗口）
- 文档向量化

**关键设计**：
- 使用PyMuPDF解析PDF，保留页面信息
- 采用RecursiveCharacterTextSplitter进行智能切分
- 支持自定义切分策略（chunk_size、overlap）

### 3.2 检索模块

**功能**：
- 向量检索（基于语义相似度）
- BM25关键词检索（基于词频）
- 混合检索（向量 + BM25）
- 重排序（Cross-Encoder）

**关键设计**：
- 混合检索使用RRF（Reciprocal Rank Fusion）融合
- 重排序使用bge-reranker-large模型
- 支持配置检索权重和阈值

### 3.3 生成模块

**功能**：
- 基于LLM生成答案
- 多轮对话管理
- 流式输出支持

**关键设计**：
- 使用LangChain封装LLM调用
- 支持多种LLM（GPT、Claude、通义千问）
- Prompt工程优化

### 3.4 向量数据库模块

**功能**：
- 文档向量存储
- 相似度检索
- 集合管理

**关键设计**：
- 使用Milvus作为向量数据库
- 支持多种索引类型（IVF_FLAT、HNSW）
- 支持多种距离度量（L2、IP、COSINE）

## 4. 性能优化

### 4.1 检索优化

- **索引优化**：使用HNSW索引提升检索速度
- **混合检索**：结合向量检索和BM25，提升召回率
- **重排序**：使用Cross-Encoder提升精度
- **缓存**：热点查询结果缓存

### 4.2 生成优化

- **流式输出**：提升用户体验
- **上下文压缩**：去除冗余信息，节省token
- **批量处理**：多文档批量向量化

### 4.3 系统优化

- **异步IO**：FastAPI异步接口
- **连接池**：数据库连接复用
- **负载均衡**：多实例部署
